Sharpened acceptance:
- The team page renders **two distinct spaces**: the existing recorded transactions
  (draft picks, from the Phase-1 `Transaction` table) and a **new "In-season moves"**
  card derived from week-over-week `team_rosters` diffs (adds / drops / retained).
- A player **drafted at the opening week** is shown as **retained**, never as an
  in-season **add** ‚Äî draft picks are not double-counted as acquisitions.
- A player who first appears **after** the opening roster week is an **add** at that
  week; a player who **disappears** between snapshots is a **drop** at the week they
  are gone. Re-add after a drop produces a fresh add (stint model).
- A season with **no week-grained roster snapshots** (can't derive churn) renders a
  `DataGap` (`roster_history_unavailable`), **never** an empty/zero "no moves" ‚Äî honesty
  rule. A season *with* snapshots but no churn renders a plain "No in-season moves".
- Roster moves are **not gated on `is_scored`** ‚Äî `team_rosters` snapshots exist for
  unscored (pre-reconstruction) seasons too, so add/drop shape is available even where
  per-player *points* are not.

## Analytics ‚Äî signature & semantics

```python
# src/ff_dashboard/analytics/transactions.py
def derive_roster_moves(session: Session, team_id: int) -> dict[str, Any] | None:
    """In-season add/drop/retain derived from week-over-week ``team_rosters`` diffs.

    Tier-1 (dashboard, read-only): the *shape* of activity only ‚Äî no nfl.com scrape,
    no exact dates / waiver-vs-FA / FAAB bids (those are UP, F-37 tier 2). Returns
    ``None`` for an unknown ``team_id`` (404).
    """
```

Return dict (‚Üí `TeamRosterMoves`):

```
{
  "team_id":     int,
  "season_year": int,
  "is_scored":   bool,                 # informational; moves are NOT gated on it
  "available":   bool,                 # False when <2 distinct roster snapshot weeks
  "roster_weeks": [int, ...],          # sorted distinct weeks with a roster snapshot
  "moves": [                           # sorted by (week, action, player_name)
    {"week": int, "player_id": int, "player_name": str|None,
     "position": str|None, "action": "add" | "drop" | "retain"},
    ...
  ],
}
```

**Derivation (stint model), per player across `roster_weeks`:**
- Build `roster_weeks` = sorted distinct `TeamRoster.week` for the team; `present[player]`
  = set of weeks the player is on the roster.
- Walk `roster_weeks` in order. For each player:
  - **enter** at week `w` ‚áî `present(w)` and (`w` is the first roster week **or** not
    `present(prev_roster_week)`). If `w` ‚âÝ first roster week ‚Üí emit **`add`** at `w`.
  - **exit** ‚áî player present at `w` but not at `next_roster_week` and `w` ‚âÝ last roster
    week ‚Üí emit **`drop`** at `next_roster_week` (the week they are gone).
  - **retain** ‚áî entered at the first roster week **and** present at the last roster week
    with no intervening exit ‚Üí emit one **`retain`** at the first roster week.
- A drop-then-readd yields a `drop` then a fresh `add` (two stints) ‚Äî honest churn.
- `available = len(roster_weeks) >= 2`. With <2 snapshots churn cannot be derived ‚Üí
  `available=False`, `moves=[]` (frontend shows the `roster_history_unavailable` gap).
  With ‚â•2 snapshots but no add/drop, `moves` carries only `retain` rows ‚Üí frontend shows
  "No in-season moves".

`acquisition_type`/`acquisition_week` on the roster row are **not** the derivation basis
(they are sparse/unreliable across seasons); the diff is the source of truth. They may be
surfaced later as corroboration (out of scope here).

## API ‚Äî route & schema (additive)

```python
# api/schemas.py
class RosterMove(BaseModel):
    week: int
    player_id: int
    player_name: str | None = None
    position: str | None = None
    action: str                      # "add" | "drop" | "retain"

class TeamRosterMoves(BaseModel):
    team_id: int
    season_year: int
    is_scored: bool
    available: bool
    roster_weeks: list[int]
    moves: list[RosterMove]
```

```python
# api/routes/teams.py
@router.get("/v1/teams/{team_id}/roster-moves", response_model=Envelope[TeamRosterMoves])
def get_team_roster_moves(team_id, session, cache) -> Envelope[TeamRosterMoves]:
    data = cache.get_or_compute(session, f"team_roster_moves:{team_id}",
                                lambda: derive_roster_moves(session, team_id))
    if data is None: raise not_found(...)        # unknown team ‚Üí 404
    return Envelope(data=TeamRosterMoves(**data), meta=build_meta(session))
```

Existing `/v1/teams/{team_id}/transactions` is **untouched**. After BUILD run
`npm run gen:api`; the drift is the new path only (additive) ‚Äî commit it.
